### 2. TUN/TAP интерфейс: `songgao/water`

| Параметр | Значение |
|----------|----------|
| **Репозиторий** | https://github.com/songgao/water |
| **Звёзды** | ⭐ 2.2k |
| **Форки** | 317 |
| **Коммиты** | 113 |
| **Релизы** | ❌ Официальных релизов нет |
| **Открытые issues** | 21 |
| **Pull requests** | 9 |

**Платформенные ограничения:**
- **Linux**: ✅ Полная поддержка TUN и TAP
- **macOS**: ⚠️ Только point-to-point TUN, имена интерфейсов генерируются автоматически (`utun<#>`), TAP не поддерживается
- **Windows**: ⚠️ Требуется внешний TAP драйвер (OpenVPN), API экспериментальный

**Известные ограничения:**
- Библиотека обёртывает только syscall'ы, использует стандартную библиотеку Go
- Может быть стабильной из-за зрелости кода, но отсутствие релизов - риск для production
- 21 открытый issue может содержать важные баги

**Рекомендация:**
- ✅ Подходит для Linux (наша основная платформа)
- ⚠️ Для production стоит мониторить issues и рассмотреть альтернативы
- 📝 Альтернатива: `github.com/kraj/nmstate` или работа с netlink напрямую

|----------|----------|

**Возможности:**

**Выводы:**

- Полностью покрывает наши потребности по управлению маршрутами
- Необходимо запускать с правами root или с capabilities

### 4. Прокси поддержка: `golang.org/x/net/proxy`

| Параметр | Значение |
|----------|----------|
| **Пакет** | `golang.org/x/net/proxy` |
| **Документация** | https://pkg.go.dev/golang.org/x/net/proxy |
| **Статус** | ✅ Часть официального Go ecosystem |

**Поддерживаемые протоколы:**
- ✅ HTTP CONNECT прокси
- ✅ SOCKS5 прокси
- ✅ Прямое соединение (Direct)
- ✅ Per-host выбор прокси

**Возможности:**
- Простой API: `proxy.Dial()`, `proxy.FromEnvironment()`
- Поддержка аутентификации для SOCKS5
- Можно комбинировать с WebSocket диалером

**Нюансы для WebSocket:**
- `gorilla/websocket` использует `net.Dialer` для подключений
- Нужно создать кастомный `Dialer` с поддержкой прокси через `golang.org/x/net/proxy`
- Для HTTP прокси: использовать `http.ProxyURL`
- Для SOCKS5: `proxy.SOCKS5()`

## 🎯 Итоговые рекомендации

### Используемые библиотеки

```go
// WebSocket транспорт
"github.com/gorilla/websocket" // v1.5.4+

// TUN интерфейс
"github.com/songgao/water" // master branch

// Управление маршрутами (только Linux)
"github.com/vishvananda/netlink"

// Прокси поддержка (клиент)
"golang.org/x/net/proxy"

// Конфигурация
"gopkg.in/yaml.v3" // или стандартный encoding/json
```

### Архитектурные решения

1. **Транспорт**: WebSocket поверх HTTP/HTTPS (TLS обязателен для production)
2. **Аутентификация**: Логин/пароль из конфига + Basic Auth при WebSocket handshake
3. **Протокол**: Бинарные WebSocket сообщения с заголовками (тип, флаги, длина)
4. **Фрагментация**: Разбивка пакетов >65KB с заголовками фрагментов
5. **Keepalive**: WebSocket ping/pong для поддержания соединения
6. **TUN**: Point-to-point интерфейс на Linux через `songgao/water`
7. **Маршруты**: `vishvananda/netlink` для управления на клиенте и сервере
8. **Прокси**: Кастомный WebSocket Dialer через `golang.org/x/net/proxy`

### Потенциальные проблемы

| Проблема | Риск | Решение |
|----------|------|---------|
| `songgao/water` без релизов | Средний | Мониторить issues, рассмотреть альтернативы |
| Требуются root права | Высокий | Документировать, использовать capabilities |
| macOS ограничения | Средний | Только Linux для первой версии |
| WebSocket через прокси | Средний | Тестировать с разными типами прокси |
| Фрагментация больших пакетов | Низкий | Реализовать с control сообщениями |
